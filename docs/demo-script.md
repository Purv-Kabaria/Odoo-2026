# AssetFlow — 6-Minute Demo Script

A judge-facing walkthrough for a screen recording. Total runtime target: **6:00**. No cold open, no throat-clearing — narration starts the instant the recording starts, over the login screen. Every "Say" line either covers a feature the brief mandates or calls out something our team added beyond it (marked inline as *"beyond the brief"* / *"our addition"*), and ties it to one concrete architecture/DB/security/robustness/scalability fact — not a separate architecture slide, folded into the feature it belongs to. Log in fresh for each role switch called out below; demo data is seeded via `pnpm db:reset:populate` (see README for the full credential table).

## What's mandated vs. what our team added

Keep this distinction explicit on camera (the script already flags each one inline) — it's the fastest way to signal "we didn't just build the checklist, we understood it well enough to extend it":

**Directly from the brief:** RBAC with non-self-elevating signup, org setup (departments/categories/employee directory), asset registration & directory, allocation with the double-allocation block, transfer requests, resource booking with half-open-interval overlap rejection, maintenance kanban with approval-gated status transitions, audit cycles with discrepancy reports, reports/analytics (utilization, maintenance frequency, most-used/idle/near-retirement, booking heatmap, department allocation summary), and the activity-log/notifications split.

**Beyond the brief, added by the team:** Asset Kits (bulk multi-asset allocation as one atomic operation), camera-based QR scanning (the brief only asked for QR *fields*, not a scan workflow), booking check-in deadlines with a grace period and auto-cancel, an LLM-assisted maintenance retirement recommendation, reusable voice-dictation input, and the Ctrl+K global command palette.

## Edge cases worth calling out verbally if time allows (skip if tight)

- **Pending approval**: a self-signed-up user can't log in until an Admin approves them — the login error message says so explicitly, not a generic "invalid credentials."
- **Kit allocation conflict**: same as the single-asset conflict banner, but for a whole kit at once — attempting to allocate a kit with even one already-held asset blocks the entire batch and names the conflicting asset, rather than partially allocating.
- **Illegal Kanban drag**: dragging a card backwards or skipping a stage is rejected client-side with a toast and the card snaps back — the drag only ever calls a legal transition endpoint.
- **Booking overlap**: shown above — this is the single most impressive "the database, not just the app, prevents this" moment, worth the extra few seconds if you have them.
- **Booking auto-cancel**: if you have a spare minute before recording, create a booking with a very short window via the API so its 15-minute check-in deadline is already close, then show the reminder notification and (if you can wait it out) the auto-cancel — otherwise just narrate the mechanism from the booking card's visible deadline.
- **Unsupported browser for voice input**: if recording in a browser without `SpeechRecognition` support, the mic button is visibly disabled with a tooltip rather than silently doing nothing.
- **QR scan denied/no camera**: the scanner modal shows a clear permission-denied or no-camera message rather than a blank or frozen video element.
- **Ctrl+K on a filtered table**: selecting a non-asset result (an employee, a department) deep-links into that entity's table pre-filtered by the same query, not just a raw navigation.

## Recording tips

- Cut the dead air entirely — the old version of this script opened on 15 seconds of nothing on screen. Don't. Start narrating over the login screen the instant the recording starts.
- Keep the browser at a standard laptop width (~1440px) for the main walkthrough; a 10-second mobile-width cutaway on the Maintenance Kanban (resize to ~390px) is a good way to prove the responsive claim without a second full pass, if you have a spare few seconds.
- If you don't have a physical QR code to scan, generate one from any asset's `qrCode` value ahead of time (any online QR generator) and display it on a second screen or phone to scan from — much smoother live than fumbling for one during the take.
- Narrate over actions rather than pausing dead air — the table above is paced assuming continuous talking; rehearse once so the "Say" lines land while the click/drag/type is happening, not after.
- Use a light-mode and dark-mode pass if you have two takes in you — the toggle is in the navbar — but pick ONE for the actual 6-minute submission to stay on time.
